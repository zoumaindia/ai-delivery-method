# Docs

`docs/` is the canonical source of truth for shared project knowledge.

## Startup order

1. `docs/README.md`
2. `docs/CURRENT.md`
3. `docs/conventions.md`
4. `docs/working-agreement.md`
5. `docs/DECISIONS.md` when rationale matters

## Startup-critical docs

- `CURRENT.md`
- `conventions.md`
- `working-agreement.md`
- `DECISIONS.md`

## Durable reference

- `overview.md`
- `TASKS.md`

## Rule

Do not leave temporary execution briefs in the startup path once they are complete.
