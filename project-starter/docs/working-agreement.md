# Working Agreement

## Collaboration rules

- Read the startup docs before meaningful work.
- Keep planning proportional to risk.
- Ask focused questions only when needed.

## Delivery rules

- No feature is done in isolation.
- Check adjacent modules when changing core objects.
- Add tests at the right layer.
- Update docs after verifying code truth.

## Definition of done

- [ ] Impact review done
- [ ] Relevant tests updated
- [ ] Builds pass
- [ ] User-facing verification done if needed
- [ ] Docs updated
