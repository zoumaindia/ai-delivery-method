# Principles

These are the default principles for AI-assisted software delivery.

## 1. Shared context beats chat context

Important project knowledge must live in docs, code, tests, and version history, not only in chat threads.

## 2. Code truth beats doc assumptions

When docs and code drift, verify the code first, then update the docs.

## 3. No feature is done in isolation

If a change touches a core object, the adjacent objects and downstream flows must also be reviewed.

## 4. AI output is draft work until verified

Generated code, plans, or tests are not complete until they have been reviewed and validated.

## 5. Tests are part of delivery

Verification is not optional polish. The right test layer should be added or updated as part of the task.

## 6. Decisions should be durable

Non-obvious product and architecture choices should be captured in a decision log.

## 7. Keep the process lean

Use the smallest amount of process that still protects quality, context, and speed.
